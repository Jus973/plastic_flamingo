Flamboyance Playground

A deliberately designed multi-page fixture site for testing Flamboyance, a tool that launches Playwright-based synthetic users against your app. Each scenario below is crafted to trigger specific frustration signals or test persona behaviors.

Main Flows (with Persona-Targeted Friction)

  • Shop Competing CTAs, confusing categories (casual_browser frustration)
  • Cart Mid purchase flow - review cart items
  • Checkout Multi-step flow with delays (frustrated_exec frustration)
  • Account Nested hidden menus, icon-only actions (non_tech_senior + accessibility_user)
  • Account Settings Basic/Advanced mode toggle, deeply nested disclosures
  • Sign Up Aggressive validation, countdown timer (anxious_newbie frustration)
  • Order Status Mobile-hostile layout, 4s delay (mobile_commuter frustration)

Stress Test Scenarios

  • Circular Navigation A ↔ B loop designed to trigger circular_navigation signal
  • Rage Decoy Non-interactive elements that look clickable (rage_click trigger)
  • Hidden Menu Collapsed nav (aria-expanded) vs visible escape hatch
  • Icon Bar Critical icon-only actions with limited labeled alternatives (accessibility_user)
  • Many Links 40+ interactive targets to stress 30-element sampling
  • Slow Page 10-second delay for testing low-patience personas
  • Edge Cases Inconsistent behaviors, dead ends, silent failures (power_user)
  • Broken Forms Subtly broken form elements for systematic testing (methodical_tester)
